# 회고 및 진화 프로토콜 (Reflection & Mutation Protocol)

오버마인드 시스템은 단위 작업, 모듈 개발, 혹은 중대한 데이터 처리를 마친 직후 **반드시** 이 워크플로우를 스스로 실행하여 경험을 영양분(유전자)으로 전환해야 합니다.

## 실행 조건
1. 새로운 기관(Mouth, Digestive System 등)의 코드 작성이 완료되었을 때.
2. 유닛(작업자)이 외부에서 데이터를 가져오거나 임무를 실패/성공했을 때.
3. 사용자가 명시적으로 회고를 요청했을 때.

## 진화 파이프라인 (수행 순서)

### 1단계: 현상 파악 (Observation)
*   방금 수행한 작업의 **목표**는 무엇이었는가?
*   수행 **결과**(성공/실패/부분 성공)는 어떠한가? 발생한 에러 로그나 비효율적인 부분은 무엇인가?

### 2단계: 인사이트 추출 (Digestion)
*   성공했다면: 왜 성공했는가? 어떤 프롬프트나 구조화 방식이 유효했는가?
*   실패했다면: 원인은 무엇인가? (예: 파싱 로직 한계, 컨텍스트 부족 등)
*   **핵심 질문**: "이 경험을 바탕으로 오버마인드의 다음 행동(규칙)을 어떻게 개선할 수 있는가?"

### 3단계: 유전자 변형 (Mutation & Memory Allocation)
*   2단계에서 얻은 교훈을 바탕으로, 기존의 시스템 규칙 파일(`.agents/workflows/` 내의 파일들)이나 코드를 직접 **수정(업데이트)**합니다.
*   예시 1: 파싱 워크플로우에 예외 처리 로직 추가.
*   예시 2: `core_directive.md`에 새로운 통제 규칙 추가.

### 4단계: 흡수 완료 보고 (Report)
*   사용자에게 다음 양식으로 짧게 보고합니다.
    *   `[진화 로그] 흡수한 경험: OOO / 변형된 유전자(수정된 파일명): OOO`
    *   어떤 인사이트를 얻어 어디를 수정했는지 명확히 밝힙니다.
